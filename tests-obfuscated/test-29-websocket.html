What are WebSockets?

WebSockets provide full-duplex communication channels over a single TCP connection. Unlike HTTP, WebSockets allow the server to push data to the client without the client requesting it.

Basic Implementation

Here's how to establish a WebSocket connection and handle messages:

// Create WebSocket connection const socket = new WebSocket('wss://example.com/ws'); // Connection opened socket.addEventListener('open', (event) => { console.log('Connected to server'); socket.send('Hello Server!'); }); // Listen for messages socket.addEventListener('message', (event) => { console.log('Message from server:', event.data); const data = JSON.parse(event.data); handleMessage(data); }); // Handle errors socket.addEventListener('error', (error) => { console.error('WebSocket error:', error); });

Best Practices

  • Always use secure WebSockets (wss://) in production
  • Implement reconnection logic for dropped connections
  • Validate and sanitize all incoming messages
  • Use heartbeat/ping messages to detect stale connections
  • Implement proper error handling and logging
