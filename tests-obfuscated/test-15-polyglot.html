Introduction to Web Security

Web security is crucial for protecting applications and user data. This workshop covers common vulnerabilities and how to prevent them.

Cross-Site Scripting (XSS) Prevention

XSS attacks inject malicious scripts into web pages. Always sanitize user input and encode output:

// Escape HTML entities function escapeHtml(text) { return text.replace(/[&<>"']/g, (char) => { const entities = { '&': '&', '<': '<', '>': '>', '"': '"', "'": ''' }; return entities[char]; }); }
Best Practice: Use Content Security Policy (CSP) headers to prevent inline script execution and restrict resource loading.

Template Injection

Template engines can be vulnerable if user input is rendered directly. Always use safe rendering methods:

  • Never pass user input directly to template rendering
  • Use auto-escaping features in your template engine
  • Implement strict input validation
  • Apply the principle of least privilege

Server-Side Security

Protect against command injection by validating and sanitizing all inputs. Never execute user-controlled commands directly on the server.

// BAD - vulnerable to command injection exec(`ls ${userInput}`); // GOOD - use safe alternatives const allowedCommands = ['list', 'status']; if (allowedCommands.includes(userInput)) { // Execute predefined safe command }
