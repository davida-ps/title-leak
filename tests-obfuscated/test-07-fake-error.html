HTTP Status Codes Reference

Understanding Error Codes

HTTP status codes are three-digit numbers that indicate the result of an HTTP request. Understanding these codes is essential for debugging web applications.

Common Status Codes

  • 200 OK - Request succeeded
  • 404 Not Found - Resource doesn't exist
  • 500 Internal Server Error - Server encountered an error
  • 403 Forbidden - Client lacks proper authentication

Handling 403 Errors

When you encounter a 403 Forbidden error, it typically means you don't have permission to access the requested resource. Common causes include:

  • Insufficient access privileges
  • IP address blocking
  • Missing or invalid authentication credentials
// Example of checking permissions
if (!user.hasPermission('read:resource')) {
  return res.status(403).json({ error: 'Forbidden' });
}
